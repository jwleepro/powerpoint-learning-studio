# PPT Coach - 요구사항 정의서

## 1. 프로젝트 개요

### 1.1 프로젝트명
**PPT Coach** - PowerPoint 실시간 가이드 & AI 피드백 시스템

### 1.2 목적
PowerPoint 초보자가 결과완료보고서를 작성할 때, 화면 위에서 단계별로 안내하고 실시간 피드백을 제공하여 고품질 문서를 완성할 수 있도록 지원

### 1.3 대상 사용자
- PowerPoint 작성 경험이 적은 직장인
- 보고서 양식을 처음 접하는 신입사원
- 체계적인 PPT 작성법을 배우고 싶은 사람

## 4. 유스케이스

### UC-01: 가이드 시작하기

**액터**: 사용자  
**사전조건**: 앱이 설치되어 있음  
**기본 흐름**:
1. 사용자가 PPT Coach 앱을 실행한다
2. 앱이 PowerPoint 설치 여부를 확인한다
3. 앱이 템플릿 선택 화면을 표시한다 (결과완료보고서, 제안서 등)
4. 사용자가 "결과완료보고서" 템플릿을 선택한다
5. 앱이 PowerPoint를 실행하고 새 문서를 생성한다
6. 앱이 가이드 패널과 오버레이를 활성화한다
7. 첫 번째 단계(슬라이드 크기 설정) 안내가 시작된다

**대안 흐름**:
- 3a. 이미 PowerPoint가 열려 있는 경우
  - 3a1. "기존 문서에서 계속" 또는 "새 문서 시작" 선택 제공
- 5a. PowerPoint 실행 실패 시
  - 5a1. 오류 메시지 표시 및 수동 실행 안내

**결과**: 사용자가 가이드와 함께 PPT 작성을 시작할 준비가 됨

---

### UC-02: 단계별 작업 수행

**액터**: 사용자  
**사전조건**: 가이드가 시작된 상태  
**기본 흐름**:
1. 앱이 현재 단계의 작업 내용을 패널에 표시한다
2. 앱이 오버레이로 작업해야 할 위치를 하이라이트한다
3. 사용자가 안내에 따라 PowerPoint에서 작업을 수행한다
4. 앱이 작업 완료 여부를 자동으로 감지한다
5. 완료 시 체크 표시와 함께 다음 단계로 안내한다
6. 사용자가 "다음" 버튼을 클릭하거나 자동 진행된다

**대안 흐름**:
- 4a. 작업이 올바르지 않은 경우
  - 4a1. 오버레이에 경고 표시
  - 4a2. 패널에 수정 방법 안내
- 6a. 사용자가 단계를 건너뛰려는 경우
  - 6a1. 건너뛰기 확인 다이얼로그 표시
  - 6a2. 나중에 돌아올 수 있음을 안내

**결과**: 한 단계의 작업이 완료되고 다음 단계로 이동

---

### UC-03: 오버레이 하이라이트 표시

**액터**: 시스템  
**사전조건**: 가이드 진행 중  
**기본 흐름**:
1. 앱이 현재 단계에서 가리킬 위치 정보를 로드한다
2. 앱이 PowerPoint 창의 위치와 크기를 계산한다
3. 앱이 투명 오버레이 창을 PPT 위에 배치한다
4. 앱이 하이라이트 영역(빨간 테두리, 반투명 배경)을 그린다
5. 앱이 화살표와 말풍선을 하이라이트 옆에 표시한다
6. 사용자 클릭이 오버레이를 통과하여 PPT에 전달된다

**대안 흐름**:
- 2a. PowerPoint 창이 이동되는 경우
  - 2a1. 실시간으로 오버레이 위치 재계산
  - 2a2. 부드러운 애니메이션으로 따라감
- 6a. 하이라이트 영역 외부 클릭 시
  - 6a1. 정상적으로 PPT에 전달
- 6b. 하이라이트 영역 내부 클릭 시
  - 6b1. PPT에 전달 + 작업 감지 트리거

**결과**: 사용자가 어디를 클릭해야 하는지 시각적으로 명확히 인지

---

### UC-04: 실시간 검증 수행

**액터**: 시스템  
**사전조건**: 사용자가 슬라이드 작업 중  
**기본 흐름**:
1. 앱이 PowerPoint 변경 이벤트를 수신한다
2. 앱이 변경된 요소의 속성을 읽는다
3. 앱이 검증 규칙(폰트, 색상, 레이아웃)을 적용한다
4. 검증 통과 시 조용히 진행한다
5. 패널의 체크리스트에 완료 표시를 업데이트한다

**대안 흐름**:
- 4a. 검증 실패 시
  - 4a1. 오버레이에 경고 아이콘 표시
  - 4a2. 패널에 문제점과 수정 방법 표시
  - 4a3. "자동 수정" 버튼 제공 (해당되는 경우)
- 4b. 여러 문제가 동시에 발견된 경우
  - 4b1. 우선순위에 따라 가장 중요한 문제부터 표시
  - 4b2. 문제 목록을 패널에서 확인 가능

**결과**: 사용자가 실수를 즉시 인지하고 수정할 수 있음
